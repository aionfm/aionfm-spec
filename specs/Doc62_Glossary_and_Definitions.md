# Document 62 – Glossary & Definitions

## Purpose

This document collects definitions of key terms, acronyms and symbols used throughout the AionFM specification.  It serves as a reference to ensure consistent understanding among stakeholders.

## Glossary

| Term | Definition |
|------|------------|
| **AionFM** | A hybrid continuous‑discrete temporal foundation model that combines continuous value patches and discrete regime tokens to produce probabilistic forecasts, scenarios and regime explanations. |
| **Value patch** | A sequence of normalized time‑series values of fixed length, representing a local window of history.  Value patches preserve numerical precision. |
| **Regime token** | A discrete symbol representing a behavioural state of the time series, derived from residual descriptors such as trend, volatility, seasonality and shocks.  Tokens are organized hierarchically into macro, meso and micro levels. |
| **Residual temporal language** | The learned language of regime tokens describing how a sequence deviates from its local baseline.  It enables transfer of temporal patterns across domains. |
| **Baseline component** | The smooth, predictable part of a time series, often modeled with trends and long‑term cycles. |
| **Seasonal component** | The repeating pattern in a time series that occurs at fixed periodicities (e.g. daily, weekly, yearly). |
| **Event or shock** | A sudden change or disturbance in a time series caused by external factors (e.g. promotions, outages, policy changes). |
| **Residual component** | The irregular part of the time series after removing baseline, seasonality and event effects.  Captures stochastic and transient behaviours. |
| **Quantile forecast** | An estimate of the value below which a specified percentage of future observations will fall.  For example, the 90th percentile predicts the value exceeded only 10% of the time. |
| **Prediction interval** | A range bounded by two quantile forecasts (e.g. 10% and 90%) that covers a specified percentage of future outcomes. |
| **Scenario path** | A sampled trajectory of future values generated by the model’s scenario generator.  Scenarios reflect potential future evolutions consistent with historical context. |
| **Regime probability** | The probability assigned by AionFM that a sequence is in a particular behavioural state at a given time.  Sum of probabilities across regimes equals 1. |
| **Retrieval module** | A subsystem that retrieves similar historical windows from a corpus to inform forecasts and explanations. |
| **Adapter** | A lightweight neural module that specializes the model for a particular domain or entity without changing the core weights. |
| **Token vocabulary** | The set of discrete regime tokens used by the model.  Tokens can be learned, hierarchical and domain‑specific. |
| **CRPS (Continuous Ranked Probability Score)** | A metric for evaluating the quality of probabilistic forecasts; lower CRPS indicates better calibrated predictions. |
| **Pinball loss** | Also known as quantile loss; used to train quantile forecasts by penalising under‑ and over‑estimation according to the quantile level. |
| **MAE (Mean Absolute Error)** | The average absolute difference between predicted and actual values. |
| **RMSE (Root Mean Square Error)** | The square root of the average of squared differences between predicted and actual values. |
| **MASE (Mean Absolute Scaled Error)** | A scale‑free error metric that divides MAE by the MAE of a naive forecast. |
| **sMAPE (Symmetric Mean Absolute Percentage Error)** | A percentage error metric that symmetrically penalises over‑ and under‑forecasts. |
| **WAPE (Weighted Absolute Percentage Error)** | The sum of absolute errors divided by the sum of actual values, often used in retail demand forecasting. |
| **CRF (Conditional Random Field)** | A statistical model used for sequence labeling.  Not used directly in AionFM but relevant for understanding regime labeling. |
| **Causal Transformer** | The decoder‑only attention architecture used in AionFM; attends only to past and current inputs to preserve causality. |
| **Context length** | The length of historical data used to condition forecasts. |
| **Forecast horizon** | The number of future time steps to predict. |
| **Calibration** | The process of ensuring that probabilistic forecasts match observed frequencies. |
| **Drift** | The change in data distribution over time, which can degrade model performance if not monitored. |
| **Adapter training** | Training of adapters on new domain data to specialise AionFM while keeping core parameters fixed. |
| **Quantisation** | The process of mapping a continuous space into discrete tokens.  In AionFM, quantisation is used to create regime tokens. |

## Symbols

| Symbol | Definition |
|-------|------------|
| `x_t` | Observed value at time t. |
| `b_t` | Baseline estimate at time t. |
| `e_t` | Normalised residual at time t. |
| `c_t` | Regime token at time t. |
| `p_k` | Value patch at index k. |
| `hat{x_{t+h}}` | Forecast value at horizon h. |
| `hat{q_{tau, t+h}}` | Quantile forecast at level tau and horizon h. |
| `eps_t` | Random noise component at time t. |
| `H` | Forecast horizon. |
| `K` | Number of patches in the context. |
| `mu_t`, `sigma_t` | Local mean and scale used for normalisation. |
